## Overview

Our color system is built on the [Gruvbox](https://github.com/morhetz/gruvbox) palette, organized into two tiers:

1. **Primitives** - Raw color values with no semantic meaning
2. **Semantic Tokens** - Purpose-driven aliases that reference primitives

Click any swatch above to copy its hex value.

## Primitives

|-------|-------|-------------|

### Accent Colors

Each accent color has four variants:

| Variant | Purpose |
|---------|---------|
| **base** | Primary use - buttons, links, interactive elements |
| **dim** | Darker variant for light mode or emphasis |
| **subtle** | Subtle backgrounds (alerts, badges, highlights) |
| **muted** | Borders, separators, hover states |

## Semantic Tokens

Purpose-driven tokens that reference the primitives. Use these in your components for consistent theming.

### Structural

| Token | Maps To | Use |
|-------|---------|-----|
| `--color-background` | `--bg` | Page background |
| `--color-foreground` | `--fg1` | Primary text |
| `--color-muted` | `--bg2` | Muted backgrounds |
| `--color-muted-foreground` | `--fg4` | Muted text |
| `--color-border` | `--bg4` | Default borders |

### Surfaces

| Token | Maps To | Use |

## Usage

### CSS Variables

```css
.my-component {
  background: var(--bg1);
  color: var(--fg1);
  border: 1px solid var(--bg4);
}

.my-component:hover {
  background: var(--bg2);
}
```

### Tailwind Classes

The semantic tokens are mapped to Tailwind utilities:

```html
<div class="bg-background text-foreground">
  <button class="bg-primary text-primary-foreground">
    Action
  </button>
</div>
```

### Status Colors

```html
<div class="bg-destructive-subtle text-destructive border border-destructive-muted">
  Error message
</div>

<div class="bg-success-subtle text-success border border-success-muted">
  Success message
</div>
```
