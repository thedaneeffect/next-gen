## Elements

### `trigger`
Spread onto a `<button>` element. Opens the alert dialog when clicked.

### `content`
Spread onto a `<dialog>` element. The main alert dialog container with `role="alertdialog"`.

### `title`
Spread onto a heading element (e.g., `<h2>`). Connected to the dialog via `aria-labelledby`.

### `description`
Spread onto a paragraph or div element. Connected to the dialog via `aria-describedby`.

### `cancel`
Spread onto a `<button>` element. Closes the dialog without taking action.

### `action`
Spread onto a `<button>` element. Confirms the action and closes the dialog.

## Controlled State

You can control the alert dialog's open state externally:

```svelte
<script lang="ts">
	import { AlertDialog } from "melt/builders";

	let isOpen = $state(false);

	const alertDialog = new AlertDialog({
		open: () => isOpen,
		onOpenChange: (open) => (isOpen = open),
	});
</script>

<button onclick={() => (isOpen = true)}>Open from outside</button>

<dialog {...alertDialog.content}>
	<h2 {...alertDialog.title}>Alert Dialog</h2>
	<p {...alertDialog.description}>Dialog is {isOpen ? 'open' : 'closed'}</p>
	<button {...alertDialog.cancel}>Close</button>
</dialog>
```

## Dialog vs Alert Dialog

| Feature | Dialog | Alert Dialog |
|---------|--------|--------------|
| Escape key closes | ‚úÖ Yes (via `closeOnEscape`) | ‚ùå No |
| Click outside closes | ‚úÖ Yes (via `closeOnOutsideClick`) | ‚ùå No |
| ARIA role | `dialog` | `alertdialog` |
| Use case | General modals | Destructive/important actions |

**Important:** `closeOnEscape` and `closeOnOutsideClick` props are **not available** for AlertDialog. It requires explicit user action via the `cancel` or `action` buttons.

Use `AlertDialog` when you need the user to explicitly acknowledge or respond to something important, such as:

- Confirming a destructive action (delete, remove)
- Displaying critical error messages
- Requiring user input before proceeding

## Configuration Options

### Scroll Behavior

Control how page scrolling behaves when the alert dialog is open:

```svelte
<script lang="ts">
	import { AlertDialog } from "melt/builders";

	const alertDialog = new AlertDialog({
		scrollBehavior: "prevent", // 'prevent' (default) or 'allow'
	});
</script>
```

- `'prevent'` (default): Prevents body scroll while dialog is open
- `'allow'`: Allows normal page scrolling

**Note:** Unlike Dialog, `'close'` is not supported for AlertDialog since it requires explicit action.

### Force Visible

Keep the dialog visible during exit animations:

```svelte
<script lang="ts">
	import { AlertDialog } from "melt/builders";

	const alertDialog = new AlertDialog({
		forceVisible: true, // Keeps dialog in DOM until CSS transitions complete
	});
</script>
```

When `true`, the dialog element remains visible (in the DOM) after setting `open = false`, allowing CSS exit animations to complete before the dialog is removed.

## Handling Actions

Both `cancel` and `action` automatically close the dialog. To perform additional operations (like deleting data), use `mergeAttrs` to compose your handler with the built-in close handler:

```svelte
<script lang="ts">
	import { AlertDialog, mergeAttrs } from "melt";

	const alertDialog = new AlertDialog();

	function handleDelete() {
		// Perform delete operation
		console.log("Item deleted");
		// Dialog closes automatically after this runs
	}
</script>

<button {...alertDialog.trigger}>Delete Item</button>

<dialog {...alertDialog.content}>
	<h2 {...alertDialog.title}>Delete this item?</h2>
	<p {...alertDialog.description}>
		This will permanently remove the item from your account.
	</p>
	<button {...alertDialog.cancel}>Keep it</button>
	<button {...mergeAttrs(alertDialog.action, { onclick: handleDelete })}>
		Yes, delete
	</button>
</dialog>
```

The `mergeAttrs` utility ensures both your `handleDelete` function and the dialog's built-in close handler run when clicked.

## Animating the Alert Dialog

The same CSS animation techniques apply as with Dialog:

```css
/* Entry animation */
dialog {
	opacity: 0;
	transform: scale(0.95);
	transition:
		opacity 0.2s ease-out,
		transform 0.2s ease-out,
		display 0.2s allow-discrete,
		overlay 0.2s allow-discrete;
}

dialog[open] {
	opacity: 1;
	transform: scale(1);
}

@starting-style {
	dialog[open] {
		opacity: 0;
		transform: scale(0.95);
	}
}

/* Backdrop animation */
dialog::backdrop {
	background: rgb(0 0 0 / 0);
	transition: background 0.2s ease-out;
}

dialog[open]::backdrop {
	background: rgb(0 0 0 / 0.5);
}

@starting-style {
	dialog[open]::backdrop {
		background: rgb(0 0 0 / 0);
	}
}
```
