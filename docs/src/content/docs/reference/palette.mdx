## Tailwind Gruvbox Edition

A complete **Tailwind-compatible color system** with 429 colors across 22 families. Replaces Tailwind's default palette while maintaining Gruvbox's warm, cohesive aesthetic.

### Color Families

**Neutrals (5 families):**
- `slate`, `gray`, `zinc`, `neutral`, `stone`

**Accents (17 families):**
- `red`, `rose`, `pink`, `fuchsia`, `purple`, `violet`, `indigo`
- `blue`, `sky`, `cyan`, `teal`, `emerald`, `green`, `lime`
- `yellow`, `amber`, `orange`

Each family has:
- **11 shades**: 50 (lightest) → 950 (darkest)
- **Muted variants** (accents only): 50% saturation for subtle backgrounds

## Usage

### Tailwind Classes

Use standard Tailwind utility classes with Gruvbox colors:

```html
<!-- Backgrounds & Text -->
<div class="bg-stone-900 text-stone-50">
  Dark background with light text
</div>

<!-- Buttons -->
<button class="bg-orange-500 text-stone-950 hover:bg-orange-600">
  Primary Button
</button>

<!-- Alerts with opacity -->
<div class="bg-red-500/10 text-red-600 border border-red-500/20">
  Error message
</div>
```

### CSS Variables

Use CSS custom properties for dynamic styling:

```css
.alert {
  background: oklch(from var(--red-500) l c h / 0.1);
  color: var(--red-600);
  border-color: oklch(from var(--red-500) l c h / 0.2);
}

.button {
  background: var(--orange-500);
  color: var(--stone-950);
}

.button:hover {
  background: var(--orange-600);
}
```

### Semantic Tokens

Optional convenience aliases are available:

```css
.card {
  background: var(--surface);
  color: var(--text);
  border: 1px solid var(--border);
}

.button-primary {
  background: var(--primary);
  color: var(--primary-text);
}
```

## Color Scale Reference

### Shade Naming

- **50**: Lightest (for light backgrounds, highlights)
- **100-400**: Light to medium-light
- **500**: Base color (main usage)
- **600-900**: Medium-dark to dark
- **950**: Darkest (for text, emphasis)

### Muted Variants

Muted colors have **50% saturation** (half the chroma of regular colors):

```html
<!-- Regular: Full saturation -->
<div class="bg-blue-500">Vibrant blue</div>

<!-- With opacity: Subtle background -->
<div class="bg-blue-500/20">Subtle blue background</div>
```

**Use muted variants for:**
- Disabled states only
- Muted colors are 40% saturation, 25% darker

**For subtle backgrounds, use:**
- Lighter shades (200, 300) OR
- Lower opacity on base colors (500/10, 500/20)

## Technical Details

### Color Generation

All colors are generated using **OKLab color space** for:
- ✅ Perceptually uniform scales
- ✅ Consistent lightness progression
- ✅ Predictable color relationships

### Gruvbox Heritage

**Original Gruvbox colors** (at shade 500):
- `red-500`: #fb4934
- `orange-500`: #fe8019
- `yellow-500`: #fabd2f
- `green-500`: #b8bb26
- `teal-500`: #8ec07c (was "aqua")
- `blue-500`: #83a598
- `purple-500`: #d3869b

**Interpolated colors** (hue mixing between Gruvbox colors):
- `rose`, `pink`, `fuchsia`: red → purple spectrum
- `violet`, `indigo`: purple → blue spectrum
- `sky`, `cyan`: blue → teal spectrum
- `emerald`: teal → green spectrum
- `lime`: green → yellow spectrum
- `amber`: yellow → orange spectrum

### Regenerating Colors

If you need to modify the color system:

```bash
# Regenerate all colors and theme mappings
pnpm gen:all

# Or individually
pnpm gen:colors  # Generate color variables
pnpm gen:theme   # Generate Tailwind theme mappings
```

Generated files:
- `docs/src/styles/tailwind-gruvbox.css` - Color definitions
- `docs/src/styles/tailwind-theme.css` - Tailwind mappings
